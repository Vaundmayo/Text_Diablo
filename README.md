C언어 기반 '텍스트 디아블로' 게임 완성하기

1. 과제 목표
본 프로젝트는 제공된 C언어 텍스트 RPG '디아블로'의 미완성 코드를 분석하고, 나머지 게임액트(Act)를 완성하는 것을 목표로 한다. 학생들은 이 과정을 통해 다음을 달성한다.
• 구조체(struct)와 2D 배열을 활용한 복잡한 게임 데이터(몬스터, 아이템, 캐릭터) 관리 기법을 익힌다.
• 기존 코드베이스(Legacy Code)를 분석하고, 정의된 함수 프로토타입에 맞춰 새로운 기능을 확장하는 능력을 배양한다.
• 파일 I/O를 통해 퀘스트 스크립트 및 몬스터 데이터를 로드하고 처리하는 방법을 학습한다.
• termios.h(Linux/macOS)와 conio.h(Windows)를 이용한 크로스 플랫폼(CrossPlatform) 터미널 제어 및 비동기 입력 처리 기법을 실습한다.

2. 과제 내용 및 방법 (Contents and Method)
제공되는 Diablo-m-test.c 파일은 게임의 기본 골격, Act 1(Quest 1~6), Act 2(Quest 1~3)
까지 구현되어 있다. 학생들은 이 코드의 구조를 파악한 뒤, 비어 있는 나머지 퀘스트 함수들을 모두 구현해야 한다.
가. 기반 코드 (Baseline)
• Diablo-m-test.c : Act 1 전체 및 Act 2 일부가 구현된 소스 코드.
• *.dat, *.qwe 파일 : 게임 실행에 필요한 퀘스트, 아이템, 마법 데이터 파일.

나. 필수 구현 기능 (Requirements)
1. Act 2 퀘스트 완성:
◦ Q2_4(), Q2_5(), Q2_6() 함수를 구현한다.
◦ Act 2의 보스(예: 듀리엘)를 Q2_6()에 구현해야 한다.
2. Act 3 퀘스트 구현:
◦ Q3_1()부터 Q3_6()까지의 모든 함수를 구현한다.
◦ Act 3의 보스(예: 메피스토)를 Q3_6()에 구현해야 한다.
3. Act 4 퀘스트 구현:
◦ Q4_1(), Q4_2(), Q4_3() 함수를 구현한다.
◦ 최종 보스(예: 디아블로)를 Q4_3()에 구현하여 게임의 엔딩을 완성한다.
4. 퀘스트 데이터 파일(.dat) 생성:
◦ Battle() 함수에 정의된 대로, 새롭게 구현하는 모든 퀘스트(Q2_4 ~ Q4_3)
에 대응하는 .dat 텍스트 파일(총 12개)을 직접 생성해야 한다.

3. 주요 구현 요건 (Cross-Platform Compatibility)
본 과제물은 Windows, Linux, macOS 세 가지 운영체제 환경에서 모두 소스 코드 수정 없이 컴파일되고 동일하게 실행되어야 한다. 제공된 래퍼(wrapper) 함수(kbhit, getch 등)를 사용해야 하며, ncurses 등 별도의 외부 라이브러리 사용을 금한다.

4. 제출 파일 (Deliverables)
아래의 파일들을 하나의 ZIP 파일로 압축하여 제출한다. (파일명: 팀장학번_이름_Diablo.zip)
1. 소스 코드 (.c): Diablo-m-test.c (모든 퀘스트가 구현된 완성본)
2. 신규 퀘스트 데이터 (.dat): Act 2(4~6), Act 3(1~6), Act 4(1~3) 시나리오 .dat 파일 총 12개.
3. 빌드 스크립트: Makefile (Linux/macOS 용)
4. 실행 보고서: README.md (OS별 컴파일/실행 방법, 구현 내용, 문제 해결 과정 명시)
